# Тольдот р. Саадьи — תולדות ר׳ סעדיה

Статический сайт на GitHub Pages для книги **«Тольдот р. Саадьи»** —
жизнеописания рабби Саадьи Либерова (авт. Мордехай-Цви Либеров, 5768/2008).

Каждая из 117 страниц показывает рядом: **изображение оригинального иврита** (справа) и **перевод на русский** (слева).

---

## Структура файлов

```
index.html          — Главная страница (оглавление + сетка всех страниц)
page.html           — Просмотр одной страницы (?p=N)
style.css           — Все стили
toc.json            — Оглавление с названиями глав
translations/
  he.json           — Текст иврита, извлечённый из PDF (стр. → текст)
  ru.json           — Русский перевод (стр. → текст), изначально пустые строки
img/
  page-001.jpg      — Изображения страниц (150 dpi, JPEG)
  page-002.jpg
  …
  page-117.jpg
original/
  Rabbi Saadya Liberow - 4 Adar I, 5768.pdf   — Исходный PDF
```

---

## Как добавлять / редактировать переводы

Откройте `translations/ru.json`. Структура файла:

```json
{
  "1": "",
  "2": "Тольдот р. Саадьи\nВехи жизни машпиа, хасида и шлиаха...",
  "3": "",
  …
}
```

- Ключи — номера страниц (строки).
- Значения — русский перевод страницы.
- Используйте `\n` для переносов строк, `\n\n` для разделения абзацев.
- Страницы с пустой строкой (`""`) покажут сообщение «Перевод не добавлен».

После редактирования сохраните файл и сделайте `git commit + push` — сайт обновится автоматически.

---

## Как редактировать оглавление

Откройте `toc.json`. Добавьте запись в массив `chapters`:

```json
{
  "page": 15,
  "title_ru": "Глава 1. Детство в Рамене",
  "title_he": "ילדות ברומן"
}
```

Поля `title_he` необязательны. Главы отображаются на главной странице в том порядке, в котором перечислены.

## Перевод через DeepL (пакетно)

Для автоматического перевода через DeepL API:

```bash
python3 translate_deepl.py --api-key YOUR_KEY
```

*(скрипт нужно написать отдельно — он читает `he.json`, отправляет тексты в DeepL и записывает результат в `ru.json`)*

## Навигация по сайту

- **Стрелки влево/вправо** (клавиатура) — перелистывание страниц.
- **Поле с номером страницы** в шапке — прямой переход.
- **Адрес** вида `page.html?p=42` — прямая ссылка на страницу.

## Деплой на GitHub Pages

1. Зайдите в Settings → Pages вашего репозитория.
2. В разделе **Source** выберите ветку `main`, папку `/ (root)`.
3. Нажмите **Save**.

Сайт будет доступен по адресу `https://<ваш-логин>.github.io/<имя-репо>/`.

Файл `.nojekyll` в корне отключает обработку Jekyll, что необходимо для правильного отображения сайта.

## Конвертация PDF (воспроизведение)

Если нужно переконвертировать PDF (например, при другом DPI):

```bash
# Установка poppler (macOS)
brew install poppler

# Конвертация в JPEG, 150 dpi
pdftoppm -r 150 -jpeg -jpegopt quality=85 \
  "original/Rabbi Saadya Liberow - 4 Adar I, 5768.pdf" \
  img/page

# Извлечение текста по страницам (Python + pdftotext)
python3 extract_text.py
```

## Технический стек

- Чистый HTML + CSS + JS — никаких фреймворков, никакого npm.
- JSON-файлы для переводов и оглавления.
- Работает полностью офлайн (открыть `index.html` напрямую).
- Lazy-loading изображений на главной; изображение текущей страницы загружается сразу.
